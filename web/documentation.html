Frequently Asked Questions

How does A¬≥E analyze my documents?
A¬≥E uses advanced AI to extract text from your uploaded documents and match it against your selected accreditation standards. Our system identifies relevant evidence and maps it to specific standards automatically.
What file formats are supported?
We support PDF, DOC, DOCX, XLS, XLSX, PPT, PPTX, and TXT files. Each file can be up to 50MB in size. For larger documents, we recommend splitting them into multiple files.
How long does analysis take?
Most documents are analyzed within 2-5 minutes. Larger files or complex documents may take up to 15 minutes. You'll receive a notification when analysis is complete.
Can I collaborate with my team?
Yes! A¬≥E supports multi-user collaboration. You can invite team members, assign standards to different departments, and track progress across your institution.
Is my data secure?
Absolutely. We use bank-level encryption for all data transmission and storage. Your documents are stored securely on AWS servers with regular backups. We never share your data with third parties.

Welcome to A¬≥E

A¬≥E (Automated Accreditation Alignment Engine) is your comprehensive solution for managing and streamlining the accreditation process. This guide will help you get started quickly.

üì§ Upload & Analysis

Document Upload Process

Our AI-powered system analyzes your documents to automatically identify and map evidence to accreditation standards.

Supported File Types

  • PDF documents (up to 50MB)
  • Microsoft Word (.doc, .docx)
  • Microsoft Excel (.xls, .xlsx)
  • Microsoft PowerPoint (.ppt, .pptx)
  • Text files (.txt)

Upload Steps

  1. Navigate to the Upload page from the main menu
  2. Drag and drop files or click to browse
  3. Select multiple files for batch processing
  4. Click "Start Analysis" to begin AI processing
  5. Monitor progress in real-time

Analysis Features

  • Text Extraction: Extracts all text content from documents
  • Evidence Identification: Identifies relevant accreditation evidence
  • Standard Mapping: Maps content to specific standards
  • Quality Scoring: Evaluates evidence strength
  • Gap Analysis: Identifies missing evidence areas

  4. Click standards to view details
  5. Select standards to include in your accreditation scope

üîß API Documentation

REST API Integration

Integrate A¬≥E with your existing systems using our comprehensive REST API. Automate workflows and sync data seamlessly.

Authentication

All API requests require authentication using a Bearer token:

Authorization: Bearer YOUR_API_KEY

Key Endpoints

  • POST /api/upload - Upload documents for analysis
  • GET /api/standards - Retrieve accreditation standards
  • GET /api/analysis/{id} - Get analysis results
  • POST /api/reports/generate - Generate reports
  • GET /api/compliance/status - Check compliance status

Integration Examples

  • Canvas LMS integration for course data
  • Document management system sync
  • Automated report generation
  • Compliance monitoring dashboards

For full API documentation, visit: api.mapmystandards.ai/docs

‚ú® Best Practices

Maximize Your Accreditation Success

Follow these proven strategies to streamline your accreditation process and ensure the best outcomes.

Document Preparation

  • Use clear, descriptive file names
  • Ensure documents are text-searchable (not scanned images)
  • Organize documents by standard or category
  • Include metadata and dates in documents
  • Review documents before uploading

Evidence Collection

  • Start collecting evidence early in the process
  • Include both quantitative and qualitative data
  • Document continuous improvement efforts
  • Maintain evidence from multiple time periods
  • Cross-reference evidence to multiple standards

Team Collaboration

  • Assign standards to appropriate team members
  • Set regular review meetings
  • Use comments and notes features
  • Track progress weekly
  • Share reports with stakeholders

Timeline Management

  • Begin preparation 12-18 months before visit
  • Complete initial gap analysis early
  • Allow time for evidence collection
  • Plan for multiple review cycles
  • Keep buffer time for unexpected needs
